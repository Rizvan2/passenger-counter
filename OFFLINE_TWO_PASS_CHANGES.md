# Офлайн-двухпроход: что изменено

## Идея
Первый проход (детекция + трекинг + ReID) теперь записывает ПОЛНУЮ траекторию каждого трека.
Второй проход в конце ролика классифицирует судьбу каждого трека, зная весь путь целиком, —
это и даёт итоговые числа. Потоковый счётчик (CrossingLineCounter) остаётся как live-превью
во время обработки; финальные числа берутся из офлайн-классификатора.

Переключается через `pc.count-mode`: `offline` (по умолчанию) или `streaming` (старое поведение).

## Новые файлы
- `model/TrackTrajectory.kt` — TrajectorySample / TrackTrajectory / TrackTrajectoryStore.
- `service/TrackFateClassifier.kt` — классификатор судьбы трека (второй проход).
- `test/.../TrackFateClassifierTest.kt` — юнит-тесты чистой логики (без ONNX/видео, запускаются).

## Изменённые файлы
- `service/AnalysisService.kt`:
  * внедрён `TrackFateClassifier` + `pc.count-mode`;
  * в цикле записывается траектория каждого ВИДИМОГО трека (pass 1);
  * после цикла в offline-режиме итоговые boardings/alightings/onboard пересчитываются
    классификатором (pass 2), с логом сравнения streaming vs offline.
- `resources/application.yml`: добавлен `pc.count-mode: offline`.

## Правила классификатора (под этот ракурс)
- Вход: устойчиво OUTSIDE (улица/проём) -> устойчиво INSIDE (ближний салон), с проходом через
  дверь ИЛИ ростом головы, и реальным смещением. Оставаться в салоне НЕ требуется (уходит в
  невидимую глубину — это норма).
- Выход: устойчиво INSIDE -> устойчиво OUTSIDE с уменьшением головы; ИЛИ «угас в двери» —
  последний надёжный кадр в DOOR, голова уменьшалась, было движение к двери (без OUTSIDE-подтв.).
- «Ушёл вглубь салона и сел» НЕ считается выходом: последняя позиция не в двери, головы не
  уменьшались.
- Стоящие/сидящие в салоне -> ONBOARD_STATIONARY (не считаются, идут в начальное заполнение).
- Пешеходы за стеклом -> PASSERBY (только OUTSIDE, ни разу не в двери) -> не считаются.
- Один track id = один человек (это обеспечивает ReID). Классификация — по трек-id.

## Проверка
- Юнит-тесты классификатора запускаются без моделей: `./gradlew test --tests '*TrackFateClassifierTest*'`.
- Полный прогон (сборка + реальный ролик) — на вашей стороне: нужны ONNX-веса и видео.
- В логах ищите строку `Offline reconciliation:` — она показывает streaming vs offline и разбивку
  по судьбам (fates), удобно для отладки на реальном ролике.
